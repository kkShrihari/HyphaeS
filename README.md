## 1. Overview

**HyphaeS** is a modular Snakemake pipeline for shotgun soil metagenomics, built for low-abundance fungal communities (~1%). It covers the full analysis from raw reads to functional annotation across 5 independent modules.

| Module | Name                  | Key Tools                                          |
|--------|-----------------------|----------------------------------------------------|
| mod1   | Quality Control       | FastQC, Trim Galore, BBDuk, Bowtie2, EukDetect, Nonpareil3, seqkit |
| mod2   | Assembly              | SPAdes (metaSPAdes)                                |
| mod3   | Taxonomy              | GTDB-Tk, Kraken2                                   |
| mod4   | Binning               | MetaBAT2, MaxBin2, DASTool                         |
| mod5   | Functional Annotation | eggNOG-mapper                                      |

**Key features:**
- One command install and setup
- Single `config.yaml` with optimal defaults and warning ranges
- 5 independent modules — each checks required inputs before starting
- Auto-logging, auto-resume, `--rerun-incomplete` built-in
- Public release ready — pip and Bioconda compatible

## 5. Quick Start

```bash
# 1. Create and enter project folder
mkdir my_soil_project && cd my_soil_project

# 2. Generate config and samples template
HyphaeS init

# 3. Install required envs
HyphaeS install-envs module --mod1

# 4. Download databases (once per machine — ~9 GB, ~30-60 min)
HyphaeS setup

# 5. Dry run to check everything
HyphaeS run mod1 --dry

# 6. Run Module 1
HyphaeS run mod1
```

## 6. CLI Commands

### Initialize project
```bash
HyphaeS init
```
Generates `config.yaml` and `samples.tsv` in the current folder with optimal defaults.

### Download databases
```bash
HyphaeS setup                          # installs to ./databases/
HyphaeS setup --db-dir /opt/databases  # installs to custom path
```
Automatically downloads, indexes, and patches `config.yaml` with correct paths. Zero manual steps.

### Check inputs
```bash
HyphaeS check mod1   # validates config + samples.tsv paths
HyphaeS check mod2   # checks mod1 outputs exist + above
HyphaeS check mod3   # checks mod2 outputs exist + above
HyphaeS check mod4   # checks mod3 outputs exist + above
HyphaeS check mod5   # checks mod4 outputs exist + above
```
Warns and stops if required inputs are missing before wasting compute time.

### Run modules
```bash
HyphaeS run mod1                # Quality Control
HyphaeS run mod2                # Assembly
HyphaeS run mod3                # Taxonomy
HyphaeS run mod4                # Binning
HyphaeS run mod5                # Functional Annotation
HyphaeS run all                 # all 5 modules sequentially

HyphaeS run mod1 --dry          # dry run — see what will execute
HyphaeS run mod1 --force        # force rerun everything
HyphaeS run mod1 --cores 32     # override thread count from config
```

> Every `HyphaeS run` automatically applies:
> `--rerun-incomplete` · `--latency-wait 60` · `--keep-going` · timestamped log

### Resume after interruption
```bash
HyphaeS run mod1    # just re-run — Snakemake detects completed steps and skips them
```

## 7. config.yaml

One file controls all 5 modules. Generated by `HyphaeS init`. Warning ranges are noted in comments.

```yaml
# ── GENERAL ──────────────────────────────────────────────────────────────────
general:
  threads: 16           # recommended: 16–144 | warning if < 8
  outdir: results/
  db_dir: databases/    # auto-filled by HyphaeS setup

# ── MODULE 1: QUALITY CONTROL ─────────────────────────────────────────────────
mod1_QC:
  min_quality: 15       # Trim Galore Phred | warning if < 10
  min_length: 50        # min read length after trim | warning if < 30
  phix_k: 31            # BBDuk kmer for PhiX | do not change
  entropy: 0.6          # low-complexity filter | range: 0.5–0.7
  host_idx: ~           # auto-filled by HyphaeS setup
  eukdetect_cfg: ~      # auto-filled by HyphaeS setup

# ── MODULE 2: ASSEMBLY ────────────────────────────────────────────────────────
mod2_assembly:
  assembler: spades
  min_contig_len: 1000  # warning if < 500

# ── MODULE 3: TAXONOMY ────────────────────────────────────────────────────────
mod3_taxonomy:
  classifier: kraken2   # options: kraken2, gtdbtk

# ── MODULE 4: BINNING ─────────────────────────────────────────────────────────
mod4_binning:
  min_completeness: 50  # DASTool cutoff | warning if < 40
  max_contamination: 10 # DASTool cutoff | warning if > 15

# ── MODULE 5: FUNCTIONAL ANNOTATION ──────────────────────────────────────────
mod5_annotation:
  eggnog_db: ~          # auto-filled by HyphaeS setup
```

- Missing columns
- Non-existent file paths

## 9. Module 1 — Quality Control

**File:** `workflow/rules/mod1_QC.smk`  
**Output:** `results/mod1_QC/`

| Step | Rule | Tool | Key parameters |
|------|------|------|----------------|
| [0]  | validate_input       | Python     | Counts R1/R2 pairs — fails if unpaired or empty |
| [1]  | fastqc_raw           | FastQC + MultiQC | Baseline QC documentation |
| [2]  | trim_galore          | Trim Galore | Q15, min 50bp, tmpdir+mv pattern (avoids naming bugs) |
| [3]  | remove_phix          | BBDuk      | k=31 hdist=1, ftm=5 for 151bp reads |
| [3b] | remove_lowcomplexity | BBDuk      | entropy=0.6 — essential for soil samples |
| [4]  | remove_host          | Bowtie2 + samtools | GRCh38, samtools extraction (not --un-conc-gz %) |
| [5]  | fastqc_clean         | FastQC + MultiQC | Post-cleaning confirmation |
| [6]  | eukdetect            | EukDetect  | Fungi screen, DB v2, ~1% expected in soil |
| [7]  | nonpareil            | Nonpareil3 | R1 only, sample-specific tmpdir (not /tmp) |
| [8]  | seqkit_stats         | seqkit     | Read count, GC%, length at raw/trimmed/clean stages |
| [9]  | retention_summary    | Python     | TSV: reads retained, PhiX%, host%, nonpareil redundancy |
| [99] | multiqc_final        | MultiQC    | Merged report across all steps and all samples |

## 10. Validation and Checks

### config.schema.yaml
Validates `config.yaml` before any module runs. Warns when values fall outside recommended ranges.

### samples.schema.yaml
Validates `samples.tsv` — checks columns, file paths, duplicates, format.

### Inter-module input check
```bash
HyphaeS check mod2
# Verifies: results/mod1_QC/04_nohost/{sample}_R1.fq.gz exists for all samples
# If missing → warning printed, pipeline stops before wasting compute
```
`HyphaeS run modX` triggers `HyphaeS check modX` automatically — no need to run manually.

## 11. Logging and Resume

### Auto-logging
No manual `tee` or `2>&1` needed — handled automatically.

| Command | Log destination |
|---------|----------------|
| `HyphaeS setup` | `databases/db_setup.log` |
| `HyphaeS run mod1` | `logs/pipeline_mod1_YYYYMMDD_HHMMSS.log` |
| Each rule | `logs/mod1_QC/{step}/{sample}.log` |

### Two log levels
- **Pipeline log** — big picture: which rules ran, timing, errors across all samples
- **Rule logs** — tool output for one sample: alignment rates, read counts, warnings

### Resume mechanism
Three layers ensure correct resume:
1. Snakemake skips rules whose output files already exist
2. `--rerun-incomplete` reruns jobs with partial/corrupted output (e.g. server crash)
3. `--keep-going` continues other samples if one fails

### Database setup resume
Uses `.done` marker files — not just file existence checks. If extraction is interrupted and produces a partial file, re-running `HyphaeS setup` detects the missing marker and reruns that step correctly.

## 12. Databases

| Database | Size | Purpose | Source |
|----------|------|---------|--------|
| PhiX (BBDuk built-in) | 0 MB | Illumina spike-in removal | BBDuk keyword `phix` — no download |
| Human GRCh38 Bowtie2 index | ~4 GB | Host read removal | Langmead AWS mirror |
| EukDetect DB v2 | ~5 GB | Fungi detection — 521k marker genes | Figshare (Lind & Pollard 2021) |
| BWA index (EukDetect) | built locally | Required by EukDetect internally | Built from EukDetect FNA |
| ete3 taxa.sqlite | ~200 MB | Taxonomy lookup for EukDetect | NCBI via ete3 |

**Total disk: ~10 GB**

All downloaded and configured automatically by `HyphaeS setup`.

## 13. Development Roadmap

### Build order
| Step | Task |
|------|------|
| 1 | `setup.py` — pip installable package |
| 2 | `cli.py` — `HyphaeS init / setup / check / run` via click |
| 3 | `templates/config.yaml` — optimal defaults + warning ranges |
| 4 | `templates/samples.tsv` — template with example rows |
| 5 | `workflow/schemas/` — config and samples validation |
| 6 | `workflow/rules/common.smk` — shared functions |
| 7 | `workflow/rules/mod1_QC.smk` — already written, integrate |
| 8 | `workflow/rules/mod2–mod5` — remaining modules |
| 9 | `workflow/envs/*.yaml` — conda environments per module |
| 10 | PyPI + Bioconda submission |

### Release path
```
pip install -e .          ← local development testing
python -m build           ← build distribution
twine upload dist/*       ← publish to PyPI
                          ← submit Bioconda recipe (1–2 week review)
conda install -c bioconda hyphae-s   ← public release
```

> All tool dependencies are already in Bioconda — no blocker for acceptance.
